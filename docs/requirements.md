# iPad テキストエディタ 要件定義書

- 作成日: 2026-07-02
- 最終更新: 2026-07-02
- ステータス: 確定(v1)

## 1. 概要

iPad 専用のテキストエディタアプリを開発し、App Store で無料公開する。
プレーンテキスト編集を高品質な基盤として最初にリリースし、その後
Markdown → コード編集 → 長文執筆支援の順に段階的に機能を拡張する。

## 2. 基本方針(確定事項)

| 項目 | 決定内容 |
|---|---|
| アプリ名 | ichiseEdit(仮称) |
| 実装形態 | ネイティブアプリ(Swift / SwiftUI) |
| 最低対応 OS | iPadOS 16.0 |
| 対応デバイス | iPad 専用(iPhone / Mac 非対応) |
| ファイル管理 | ファイル App 連携(iCloud Drive・ローカル・他プロバイダ) |
| 配布 | App Store で一般公開 |
| 収益化 | 完全無料(課金なし) |
| 表示方向 | 横書きのみ(縦書きは対応しない) |
| リリース戦略 | プレーンテキスト基盤の MVP → 段階拡張 |

## 3. リリースロードマップ

### Phase 1 — MVP(v1.0): プレーンテキストエディタ

高品質なプレーンテキスト編集 + ファイル App 連携を完成させて公開する。

### Phase 2(v1.1): Markdown 対応

シンタックスハイライトとプレビューを追加する。

### Phase 3(v1.2): コード編集対応

プログラミング言語のハイライト・行番号など開発者向け機能を追加する。

### Phase 4(v1.3): 長文執筆支援

文字数カウント強化・集中モードなど執筆向け機能を追加する。

### Phase 5: ISLISP マクロ(2026-07-04 追加)

エディタ拡張用のマクロ言語として ISLISP サブセットを組み込む。
詳細は [macro-requirements.md](macro-requirements.md) を参照。

**リリース計画の変更**: Phase 1〜4 は段階リリース予定だったが、提出前に全フェーズの
実装が完了したため、**Phase 5(マクロ)まで含めて v1.0 として提出する**。

## 4. 機能要件

### 4.1 Phase 1(MVP)— 必須

**ファイル操作**
- ドキュメントベースアプリ(`DocumentGroup` / ドキュメントブラウザ)として実装
- ファイル App(iCloud Drive、ローカル、サードパーティプロバイダ)からの開く/保存
- 新規作成、名前変更、複製、削除(ドキュメントブラウザ標準機能)
- 自動保存(明示的な保存操作を不要にする)
- 対象拡張子: `.txt`(`.md` などは Phase 2 以降で追加)
- 文字コード: UTF-8 のみ(読み書き)。BOM 付き UTF-8 の読み込みに対応。
  Shift_JIS 等のレガシーエンコーディングは対応しない
- 改行コード: LF / CRLF を保持して読み書き

**編集**
- 日本語入力(IME・インライン変換)の正常動作
- 取り消し / やり直し(Undo / Redo)
- 検索・置換(大文字小文字の区別オプション付き)
- テキスト全選択・カット・コピー・ペースト(標準動作)

**表示・UI**
- フォントサイズ変更
- ライト / ダークモード対応(システム設定に追従)
- 行の折り返し表示
- 文字数・行数の表示

**iPad 最適化**
- 外部キーボードショートカット(⌘N 新規、⌘F 検索、⌘Z/⇧⌘Z、⌘S 等)
- Split View / Slide Over / Stage Manager での動作
- 同一アプリの複数ウィンドウ(マルチシーン)対応
- ソフトウェアキーボード表示時のレイアウト調整(編集領域が隠れない)

### 4.2 Phase 2 — Markdown

- `.md` / `.markdown` の対応
- Markdown シンタックスハイライト(見出し・強調・リンク・コードブロック等)
- プレビュー表示(エディタとの分割表示 or 切り替え)
- 編集支援: リスト自動継続、ツールバーからの記法挿入

### 4.3 Phase 3 — コード編集

- 主要言語のシンタックスハイライト(対象言語は Phase 3 着手時に選定)
- 行番号表示
- 等幅フォント選択
- インデント設定(タブ / スペース、幅)
- 自動インデント

### 4.4 Phase 4 — 長文執筆支援

- 詳細な文字数カウント(選択範囲、原稿用紙換算など)
- 集中モード(現在の段落以外を減光する等)
- セッション目標(文字数目標)→ 着手時に再検討した結果、v1 では**見送り**
  (統計ポップオーバーで代替可能なため。要望があれば将来バージョンで検討)

## 5. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | 1MB 程度のテキストでもスクロール・入力にもたつきがないこと |
| データ保全 | クラッシュ・強制終了時も編集内容を失わない(自動保存+ドキュメントの状態復元) |
| オフライン | ローカルファイルは完全オフラインで編集可能。iCloud はオフライン時も編集でき、接続時に同期 |
| アクセシビリティ | Dynamic Type、VoiceOver の基本対応 |
| ローカライズ | 日本語・英語の 2 言語(UI 文言) |
| プライバシー | ユーザーデータの外部送信なし。解析 SDK も入れない(プライバシーポリシーを簡潔にできる) |

## 6. 技術構成(案)

| 項目 | 内容 |
|---|---|
| 言語 / UI | Swift + SwiftUI(エディタ本体は `UITextView`(TextKit 2)を `UIViewRepresentable` でラップ) |
| ドキュメント | `DocumentGroup` + `FileDocument`(値型。バインディング経由の変更で自動保存が働くため実装が簡潔。パフォーマンス上の問題が出た場合のみ `ReferenceFileDocument` へ移行を検討) |
| プロジェクト生成 | XcodeGen(`project.yml` が原本。`.xcodeproj` は `xcodegen generate` で再生成) |
| ハイライト | Phase 2 以降。Markdown は自前実装 or ライブラリ、コードは tree-sitter 系ライブラリを候補として比較検討 |
| 最低 OS | iPadOS 16.0 |
| 依存管理 | Swift Package Manager |
| CI | 当面なし(必要になったら GitHub Actions を検討) |

エディタ本体に SwiftUI `TextEditor` を使わないのは、検索ハイライト・
シンタックスハイライト・キーボード制御など将来要件に対する拡張性が
不足するため。最初から TextKit ベースで作る。

注意: iPadOS 16 対応のため、iOS 17 以降でしか使えない API
(例: `UITextView.textLayoutManager` の一部新機能、最新 SwiftUI API)は
可用性チェック(`if #available`)付きで使うか避ける。
`UITextView` は iOS 16 以降デフォルトで TextKit 2 で動作するが、
`layoutManager` プロパティに触れると TextKit 1 にフォールバックするため、
実装時はアクセスしないよう注意する。

## 7. App Store 公開に必要な準備

- Apple Developer Program への加入(年間 $99)
- アプリ名(App Store 上の表示名)の決定 → §9
- アプリアイコン
- スクリーンショット(iPad 13" / 11")
- プライバシーポリシー(データ収集なしのため簡潔なもので可。掲載先 URL が必要)
- App Store 説明文(日 / 英)
- 審査対応(Guideline 4.2「最小限の機能」対策として、MVP でも検索・置換や
  キーボードショートカット等、標準メモ帳との差別化点を備えておく)

## 8. スコープ外(やらないこと)

- 縦書き表示
- iPhone / Mac 対応
- 課金・広告
- 独自クラウド同期(iCloud Drive のファイル同期に委ねる)
- リアルタイム共同編集
- プラグイン機構(ネイティブコードによる拡張)
  ※ ISLISP マクロによる拡張は Phase 5 として対応(macro-requirements.md)

## 9. 確認済み事項と今後の準備タスク

以下は 2026-07-02 に確認済み。

1. **アプリ名**: ichiseEdit(仮称)。Bundle ID は Xcode プロジェクト作成時に決定
   (例: `com.ichise.ichiseedit`)。正式名称は App Store 提出前に最終決定する。
2. **最低対応 iPadOS**: 16.0。
3. **文字コード**: UTF-8 のみ(§4.1 に反映済み)。

Developer Program 未加入に伴う進め方:

- 開発中は無料の Personal Team で実機テスト可能
  (署名の有効期限が 7 日のため、週 1 回程度の再インストールが必要。
  アプリ内データ=ファイル App 上のテキストは消えない)。
- App Store 提出および TestFlight 利用には Developer Program 加入(年間 $99)が必須。
  リリースが視野に入った段階(Phase 1 完成の目処が立った頃)に加入すればよい。
